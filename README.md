## API с функциями блокировки пользователей, маскированию заблокированных слов и отправки сообщений;

### KafkaConfig содержит стандартное наполнение с конфигурацией работы Кафки и созданием топиков по умолчанию.
### TestController содержит эндпоинты, с помощью которых отправляем сообщения и тестируем блокировку пользователей и сообщения
### BannedWordProcessor процессор с логикой блокировки слов (добавление данных в store)
### BlockEventProcessor процессор с логикой блокировки пользователей (добавление данных в store)
### MessageFilteringProcessor процессор с логикой проверки в store'ах данных и дальнейшей фильтрацией сообщений
### ProcessTopologyBuilder фактически создание топологии с использованием StreamBuilder, добавлением Serdes и вызовом process методов при триггере того или иного топика

## Запуск приложения:
### 1) docker-compose up -d --build используем команду в терминале;
### 2) переходим на localhost:8080 для графического отображения работы kafka с помощью kafka-ui;
### 3) отправляем запросы с помощью postman или аналогов на соответствующие эндпоинты;
### 4) проверяем работоспособность, проваливаясь на kafka-ui в topics -> ищем нужный нам топик -> messages;

## Тестовые данные можно придумать самостоятельно, нужно лишь соблюдать последовательность:
### 1) отправляем /send-text с body: senderId, recipientId и text. Удостовряемся, что сообщение отправилось и находится в messages топик;
### 2) блокируем слово /add-banned-word с body: word. Удостоверяемся, что слово добавилось в banned_words топик;
### 3) повторно отправляем /send-text, где в text должно содержаться слово, которое добавили в banned_words. Переходим в filtered_messages, чтобы удостовериться, что текст замаскировался;
### 4) блокируем пользователя /block-user с body: blockerId, blockedId. Переходим в топик blocked_users, удостоверяемся, что пара добавилась;
### 5) повторно отправляем /send-text, где senderId = blockedId, а recipientId = blockerId. Переходим в messages топик, чтобы удостовериться, что сообещние не отправилось. (на данном этапе желательно, чтобы text не содержал запрещенных слов, т.к. в filtered_messages оно отправится именно по этой причине)
